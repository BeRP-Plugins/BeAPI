---
title: Modal
---

The primary interaction point for handing schemas and documents.

:::info
See [examples/databasing](/docs/examples/databasing) for getting started with databasing.
:::

## Constructor
```js
new Modal(name, schema, scoreboardName);
```
| PARAMETER | TYPE | OPTIONAL | DEFAULT | DESCRIPTION |
| :-------: | :--: | :------: | :-----: | :---------: |
| name | [string](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String) |  ❌   | `undefined`  | Name of model (used to reference serialized data for fetching). |
| schema | [Schema](/docs/classes/schema) |  ❌   | `undefined`  | Blueprint for modal. |
| scoreboardName | [string?](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String) |  ✔️   | `BeAPI__DATABASE`  | Name of scoreboard documents for this modal should go under. |

## Properties

### schema `READONLY`
Parent schema blueprint.

**Type:** [Schema](/docs/classes/schema)
___
### name `READONLY`
Schema name used to categorize documents.

**Type:** [Schema](/docs/classes/schema)
___
### scoreboardName `READONLY`
Scoreboard name in which schema saves documents to.

**Type:** [Schema](/docs/classes/schema)
___

___

### deleteAll
Deletes all documents from database.

:::warning
This is pretty dangerous and irreversible... Be careful!
:::

```js
deleteAll(data);
```

#### Returns
void
___
